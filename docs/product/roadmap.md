# Roadmap

## Phase 1: Single-Player Profile Intelligence

Goal: Help a user understand themselves and build a high-quality structured profile.

Focus areas:

- AI-guided profile discovery
- Structured profile schema exploration
- Attribute confidence and versioning
- Privacy classification
- User-facing profile summaries

## Phase 2: Simulated Compatibility Evaluation

Goal: Validate deterministic matching with synthetic or private candidate data before building a marketplace.

Focus areas:

- Relationship context configuration
- Hard filters and soft scoring
- Explanation quality
- Offline evaluation
- Cost and latency constraints

## Phase 3: Small Controlled Marketplace

Goal: Introduce real candidate discovery with strong consent and limited exposure.

Focus areas:

- Candidate visibility policy
- Introduction consent
- Feedback loops
- Match quality metrics
- Abuse prevention

## Phase 4: Multi-Context Platform

Goal: Reuse the same compatibility engine across multiple relationship types.

Focus areas:

- Context-specific schemas
- Marketplace supply-demand balancing
- Configurable ranking policies
- Segmented privacy rules
- Partner or community integrations

## Implementation Rule

Do not start building feature code until the architecture, domain model, and first relationship context configuration are intentionally chosen.
